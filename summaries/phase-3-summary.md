# Phase 3 要約：セキュリティ強化 (Security Hardening)

Phase 3では、アプリケーションの脆弱性を具体的に修正し、実務レベルの堅牢性を備えたシステムへと強化しました。

## 学んだこと
- **多層的な防御**: 入力チェック、データの暗号化、通信の保護など、複数の観点からセキュリティを強化する方法を学びました。
- **バリデーションの実装**: ユーザーから渡されるデータが正しい形式か、処理を始める前に厳格にチェックする重要性を学びました。
- **安全なパスワード運用**: パスワードをそのまま保存せず、ハッシュ化することで漏洩時のリスクを最小化する手法を実践しました。
- **セキュアなクエリ発行**: SQLインジェクションを根本から防ぐために、プレースホルダを使用する習慣を身につけました。
- **トークンベースの認証**: ログイン状態を安全に管理し、特定のページや機能へのアクセス権を制御する仕組み（認可）を構築しました。

## 初めて登場した用語・概念
- **Zod**: TypeScriptと相性の良いスキーマ検証ライブラリです。「期待通りのデータか」を簡単にチェックできます。
- **Argon2**: 現在最も推奨されているパスワードハッシュ化アルゴリズムの一つです。計算コストを高く設定できるため、解析攻撃に強いのが特徴です。
- **Hashing (ハッシュ化)**: 元のデータを、特定の計算によって全く別の文字列（ハッシュ値）に変換することです。元に戻すことができない「一方通行」の変換です。
- **Placeholder (プレースホルダ)**: SQL文の中で、後から実際の値を当てはめるための「予約席（`?`）」のことです。
- **JWT (JSON Web Token)**: 情報を安全に受け渡すための、デジタル署名付きのオープンな標準規格です。ログイン証明書としてよく使われます。
- **Authentication (認証)**: 「誰であるか（本人確認）」を確認することです（例：ログイン）。
- **Authorization (認可)**: 「何ができるか（権限確認）」を確認することです（例：管理者のみアクセス可能）。
- **@fastify/jwt**: FastifyでJWTを簡単に扱うためのプラグインです。
