# Phase 1 要約：環境構築と現状把握

Phase 1では、ハンズオンの土台となる環境を構築し、あえて「脆弱で構造の乱れたコード」を作成することで、現状の問題点を把握しました。

## 学んだこと
- **開発環境のセットアップ**: Node.jsとTypeScriptを使用したプロジェクトの初期化方法を学びました。
- **Webサーバーの構築**: Fastifyを使用して、基本的なAPIエンドポイントを作成しました。
- **データベースの利用**: SQLite3を使用して、インメモリデータベース（実行中のみ保持されるDB）を構築しました。
- **問題の可視化**: セキュリティ対策やコードの整理が行われていない「泥団子状態」のコードが、どのようなリスク（SQLインジェクションなど）を抱えているかを確認しました。

## 初めて登場した用語・概念
- **Node.js**: JavaScriptをブラウザの外（サーバー上など）で動かすための実行環境です。
- **npm (Node Package Manager)**: ライブラリ（便利なプログラムの集まり）を管理・インストールするためのツールです。
- **TypeScript**: JavaScriptに「型（データの種類）」の概念を追加した言語です。エラーを未然に防ぎやすくなります。
- **Fastify**: Node.jsで動作する、高速で効率的なWebフレームワーク（APIサーバーを作るための枠組み）です。
- **SQLite3**: 設定が簡単で軽量な、ファイルベースのデータベース管理システムです。
- **Big Ball of Mud (巨大な泥団子)**: 全体的な構造がなく、各部分が複雑に絡み合って理解や修正が困難になったソフトウェアの状態を指す言葉です。
- **SQLインジェクション**: ユーザー入力に悪意のあるプログラムを混ぜることで、データベースを不正に操作する攻撃手法です。
- **平文保存**: パスワードなどの機密情報を、暗号化せずにそのままのテキストで保存することです。
