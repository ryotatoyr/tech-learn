# CLI操作ハンズオン 要約

このハンズオンでは、PowerShell（Windows環境）における基本的なファイル操作から、パイプラインを活用した検索・フィルタリング、Gitの基本的なワークフローまでを体験しました。

## 学んだこと

### 1. 基本的なファイル・ディレクトリ操作
- **概要**: CLIを用いたディレクトリ作成、ファイル作成、書き込み、コピー、削除の基礎。
- **実装**: `mkdir`, `ni`, `echo`, `cat`, `cp`, `rm` などのコマンドを使用。
- **効果**: GUIを使わずに、コマンドラインだけで効率的にファイルシステムを操作できるようになった。

### 2. パイプラインと検索・フィルタリング
- **概要**: 複数のコマンドを組み合わせ、大量のデータから必要な情報を抽出する。
- **実装**: `Select-String` による文字列検索、パイプライン(`|`)によるコマンド連結、ワイルドカード(`*`)によるファイル指定。
- **効果**: ファイルの内容やファイル名のリストから特定の条件に一致するものを即座に見つけ出せるようになった。

### 3. コマンドラインによるGit操作
- **概要**: コマンドラインのみでリポジトリの初期化からコミット、履歴確認までを行う。
- **実装**: `git init`, `git status`, `git add`, `git commit`, `git log` を使用。
- **効果**: GUIツールに頼らず、開発ワークフローに不可欠なバージョン管理の基本操作をCLIで完結させられるようになった。

## 初めて登場した用語・概念
- **Pipeline (|)**: 前のコマンドの出力を次のコマンドの入力として渡す仕組み。
- **Select-String**: ファイル内のテキストを検索するPowerShellのコマンド（Linuxのgrepに相当）。
- **Wildcard (*)**: 任意の文字列にマッチする特殊文字。
- **Staging (git add)**: 変更内容を次のコミットの対象として登録する作業。
