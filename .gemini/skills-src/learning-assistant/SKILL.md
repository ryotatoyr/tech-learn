# learning-assistant

このスキルは、学習者が効率的に学習を進められるよう、事務的・補助的な作業をサポートします。

## ワークフロー

### 1. 進捗管理とコミット
タスクが完了したら、以下の手順を自動的に実行します。
1.  該当するディレクトリの `TODO.md` を更新（[ ] を [x] に変更）します。
2.  変更を git に commit します。コミットメッセージは「Phase X: [タスク名] の完了」のように簡潔にします。
3.  コミットが3つ以上溜まったら、学習者に確認した上で `git push` を提案します。

### 2. 学習内容の要約
Phaseや大きな区切りが完了した際、学習内容を要約して `summaries/[学習ディレクトリ名]/` ディレクトリに保存します。

**要約フォーマット:**
1.  **タイトル**: `# Phase X 要約：[フェーズ名]`
2.  **導入**: フェーズの目的を1-2文で簡潔に記載。
3.  **学んだこと**: 主要なトピックごとに以下の3項目を記述。
    - **概要**: 何を目的としたタスクか
    - **実装**: 具体的にどのようなツールや手法を用いたか
    - **効果**: それによってどのような改善や成果があったか
4.  **初めて登場した用語・概念**: `## 初めて登場した用語・概念` セクションに簡潔な説明を記載。

**ファイル名:** `summaries/[学習ディレクトリ名]/phase-X-summary.md` (例: `summaries/backend-performance-handson/phase-1-summary.md`)

### 3. エラー対応
学習者がエラーに遭遇した場合、以下のステップで支援します。
1.  分析・修正手順の提示（修正は学習者が実施）。
2.  必要に応じて `docs/knowledge-base.md` への追記提案。
